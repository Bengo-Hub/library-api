# Support

This document outlines support channels for the Library Service.

## Issue Tracker

- Use the repository issues to report bugs or request features.
- Provide tenant ID, environment, reproduction steps, and logs (sanitised).
- Label issues appropriately (`bug`, `enhancement`, `ops`, etc.).

## Internal Channels

- Slack: `#bengobox-library` (engineering), `#library-support` (ops).
- Weekly triage call: Wednesdays 14:00 EAT.
- Escalations: Opsgenie on-call schedule for critical incidents.

## External Partners

- Email `support@bengobox.com` with SLA classification.
- Emergency hotline available via partnership agreements.

## Documentation

- `docs/plan.md` – roadmap, scope, phased plan.
- `docs/architecture.md` – service architecture + circulation engine.
- `docs/erd.md` – data model reference.
- `docs/api-reference.md` – REST endpoint reference.
- Additional ADRs and integration guides under `docs/`.

## SLAs

- Target uptime: 99.9%.
- P1 (system down): 1-hour response, continuous updates.
- P2 (degradation): 4-hour response, updates every 4 hours.
- P3 (general): 2 business day response.

Thank you for helping improve the Library Service. Detailed reports enable faster resolutions.
